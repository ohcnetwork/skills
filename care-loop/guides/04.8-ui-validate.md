## App runnability

Live mode needs the dev server on **:4000** against the backend on **:9000**. The backend should already be running (the loop's `preflight.sh` verified it); surface a `BLOCKED` if :9000 is down.

**A :4000 that's already up may belong to another worktree — never validate against the wrong tree.** Worktree loops run in parallel and the user's main checkout commonly serves Vite on :4000, so a bare "port is up → reuse it" check can screenshot a _different_ branch's code and make every finding meaningless. Before reusing a running :4000, confirm it serves **this** worktree (the listener's cwd must be inside `<worktree>`); if it belongs elsewhere, start your own server on a free port and use that:

```bash
DEV_PORT=4000
serves_this_tree() {   # true if the :$1 listener's process cwd is inside this worktree
  local pid; pid=$(lsof -nP -iTCP:"$1" -sTCP:LISTEN -t 2>/dev/null | head -1) || return 1
  [ -n "$pid" ] || return 1
  local cwd; cwd=$(lsof -a -p "$pid" -d cwd -Fn 2>/dev/null | sed -n 's/^n//p' | head -1)
  case "$cwd" in "$PWD"|"$PWD"/*) return 0;; *) return 1;; esac
}
if curl -sf "http://localhost:$DEV_PORT" >/dev/null 2>&1 && ! serves_this_tree "$DEV_PORT"; then
  DEV_PORT=4001; while lsof -nP -iTCP:"$DEV_PORT" -sTCP:LISTEN -t >/dev/null 2>&1; do DEV_PORT=$((DEV_PORT+1)); done
  echo "STATUS: :4000 serves another worktree — starting our dev server on :$DEV_PORT" >> "<run-dir>/agents/care-ux-validator.log"
fi
# start our own only if nothing of OURS is already serving the chosen port
if ! { [ "$DEV_PORT" = 4000 ] && serves_this_tree 4000; }; then
  echo "HEARTBEAT $(date -u +%Y-%m-%dT%H:%M:%SZ)" >> "<run-dir>/agents/care-ux-validator.log"
  npm run dev -- --port "$DEV_PORT" >> "<run-dir>/agents/dev-server.log" 2>&1 &
  DEV_SERVER_PID=$!
  for i in $(seq 1 30); do
    sleep 1
    curl -sf "http://localhost:$DEV_PORT" >/dev/null 2>&1 && break
  done
fi
```

Navigate the browser session at `http://localhost:$DEV_PORT`. Kill the dev server (`kill $DEV_SERVER_PID`) if _you_ started it — a validator that dies mid-run leaks the server it spawned, so [00-resume.md](./00-resume.md) checks for and reaps an orphaned `dev-server.log` process on re-entry. Leave a server alone if it was already serving this worktree.

Run the entire live browser session under **`pw-lock.sh`** — the backend/DB is a shared singleton across concurrent worktree loops:

```bash
"$SKILL_DIR/pw-lock.sh" -d "<run-dir>" -- <browser-session-commands>
```

Emit `HEARTBEAT` to `agents/care-ux-validator.log` before starting the browser session (a Playwright session can block for minutes; the heartbeat keeps the watchdog quiet).
